# 스마트 포도 선별기 현장 도입 및 개발 환경 기술서

본 문서는 '스마트 포도 선별기' 프로젝트를 실제 현장(공장, 농장 등)에 안정적으로 배포하고 유지/보수하기 위한 하드웨어, 소프트웨어, 배포 프로세스의 기준 규격을 정의합니다.

## 1. 하드웨어(Hardware) 환경 명세

### 1-1. 메인 프로세서 (제어 및 UI 렌더링)
- **보드 규격**: Raspberry Pi 3 Model B (또는 4B 호환 권장)
- **메모리(RAM)**: 최소 1GB 이상 (권장 2GB 이상)
- **저장매체**: MicroSD 카드 Class 10 이상 (용량 16GB 이상 권장)
- **디스플레이**: 7인치 터치스크린 диспл (해상도 800x480 또는 1024x600 권장)
- **네트워크**: 이더넷 연결 또는 신뢰도 높은 무선 Wi-Fi 허브 구비 (OTA 업데이트 시 필수 요구)

### 1-2. 서브 컨트롤러 (데이터 수집 및 하드웨어 통신 제어)
- **보드 규격**: Arduino Mega 2560 (CH340 또는 정품 USB 직렬 드라이버 칩셋 탑재형)
- **통신 연결**: USB to Serial 케이블 (A-to-B 타입), 외부 전원 공급 어댑터 권장 (9V 1A)
- **I/O 인터페이스**: 포도 트레이에 부착된 12개 체널 ADC(로드셀 앰프, HX711 등) 구비 및 LED 알림 출력 핀 할당 (12개)

## 2. 소프트웨어(Software) 환경 명세

### 2-1. 임베디드 OS (라즈베리파이)
- **운영체제**: Raspberry Pi OS Lite (64-bit 권장, Desktop 환경 제외 CLI 버전 적용 필요)
- **GUI 그래픽 백엔드**: X Server (`xinit`, `x11-xserver-utils`) – 창 관리지(Window Manager) 없이 Kiosk 모드로 단도 구동 제어
- **입력 도구**: 터치 이벤트 적용 시 추가 마우스 커서 숨김 모듈 탑재 (`unclutter`)

### 2-2. 애플리케이션 프레임워크 (파이썬)
- **언어 버전**: Python 3.9 이상
- **GUI 렌더링**: PyQt5 (`pyqt5` 패키지) - 하드웨어 가속 및 고해상도 안티앨리어싱 스케일 환경 구성
- **시리얼 통신**: PySerial (`pyserial` 커넥터 모듈, 내부 Baudrate 고정 115200bps 설계)
- **데이터 구조화 모듈**: `itertools` 커버 모듈을 통한 최적화 브루트 포스 콤비네이션 연산 대응
- **내장 폰트 리소스**: 라이선스가 자유로운 '나눔바른고딕'(`NanumBarunGothic.ttf`) 혹은 이와 상응하는 굵은 고딕체 필요.

### 2-3. 형상 관리 (버전 관리 시스템)
- **로컬 패키지**: `git` 패키지 설치 필요. 권한 문제 해결을 위해 특정 사용자로 폴더 접근 권한(Ownership) 맞춤 설정

## 3. 현장 배포 가이드 및 셋업 절차 (Deployment Process)

### Step 1: 기기 OS Flashing 및 네트워크 셋업
1. Raspberry Pi OS Lite를 SD 카드에 퍼징. (Raspberry Pi Imager의 고급 셋업 중 `SSH 활성화` 및 `Wi-Fi` 미리 설정 포함)
2. 첫 구동 시, 터미널 로그인을 수행하고 `sudo apt update && sudo apt upgrade -y` 로 커널 및 베이스 패키지 갱신.

### Step 2: 자동 Kiosk 인스톨 구성
1. 제공된 `setup_kiosk.sh` 을 라즈베리파이의 작업 디렉토리에 복제합니다.
   > **작업 권장 디렉토리 경로:** `/home/pi/포도선별기`
2. `sh setup_kiosk.sh` 명령어를 통하여 X 서버, PyQt5, 파이썬 기반 통신 드라이버를 자동 설치하도록 지시합니다. (소요 시간 5분 내외)

### Step 3: 부팅 콘솔 환경 통제
1. `sudo raspi-config` 명령어 실행
2. `<1 System Options>` -> `<S5 Boot / Auto Login>` 메뉴 진입
3. `<B2 Console Autologin Text console, automatically logged in as 'pi' user>` 활성화 처리
   > 이 셋업은 시스템이 부팅하자마자 즉시 X 서버 이벤트를 Hookup 하여 파이썬 앱을 실행하게 만듭니다.

### Step 4: 원격 업데이트 (OTA) 설정 유의점
메인 애플리케이션(`main.py`)은 시작 루틴으로 `check_ota_update()` 함수를 호출합니다. 이 절차를 안전히 다루기 위해선:
- 선별기 애플리케이션 코드가 위치한 로컬 포인터가 실제 Git 브랜치(origin/main)를 Track 하고 있어야 합니다.
- SSH Key 혹은 GitHub PAT(Personal Access Token) 형태의 Credential이 해당 라즈베리파이에 저장되어 패스워드 입력 과정을 건너뛰도록 구성해야 현장에서 안전하게 팝업 업데이트 안내를 받을 수 있습니다.

## 4. 디버깅 및 트러블슈팅

1. **시리얼 포트 인식 오류** 
   - 증상: 아두이노 USB를 꽂았음에도 화면에서 연결 실패 문구 렌더링
   - 대처: 터미널에서 `ls /dev/ttyA* /dev/ttyU*` 입력 후 노출 포트명을 메인 소스코드 연결 목록(`ports=[]`) 풀에 수동 편입. 장치 권한 이슈(`sudo usermod -a -G dialout pi`) 수행 요망.
2. **최적 조합 화면 출력이 현저히 느려질 경우 (병목)**
   - 대처: 목표 무게 여유 오차(±50g) 및 최대 조합 한계를 4개 초과 구성(예: 8개 이상)으로 셋업 시 라인 연산에 스레드 프리징 유발 가능성이 있습니다. 라즈베리파이3B 의 성능 상 한계이므로 `최적화 조합`은 5 내외로 유지하도록 현장 안내.
